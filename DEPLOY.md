# Deploy this profile README

The three visuals (contribution graph, ASCII portrait, neofetch info card) are
already generated and committed. To make them show on your GitHub profile:

## 1. Create your profile repo
GitHub shows the README of a repo **named exactly like your username** on your
profile. Create a public repo called **`ritessshhh`** (i.e. `ritessshhh/ritessshhh`).

## 2. Push these files
```bash
git init
git add .
git commit -m "profile README with live stats + ascii portrait"
git branch -M main
git remote add origin https://github.com/ritessshhh/ritessshhh.git
git push -u origin main
```

## 3. Add your token (for PRIVATE stats + lines of code)
Without a token the daily Action still runs using GitHub's built-in token, but it
only sees **public** activity. To include private repos and a full lines-of-code
count:

1. Create a token: GitHub → Settings → Developer settings → **Personal access
   tokens (classic)** → Generate new token → check the **`repo`** scope.
2. Add it to this repo: repo → Settings → Secrets and variables → **Actions** →
   New repository secret → name it **`GH_STATS_TOKEN`**, paste the token.
3. (Recommended) GitHub → Settings → Profile → enable
   **"Include private contributions on my profile"** so private days count in the
   graph too.

## 4. Run it once
Repo → **Actions** tab → enable workflows → "Update profile art" → **Run
workflow**. It refreshes `contrib-heatmap.svg`, `info-card.svg`, and
`data/stats.json` daily from then on. The lines-of-code line appears on the info
card after this first tokened run.

## Regenerating the ASCII portrait (only if you change your photo)
The portrait is static — not regenerated by the Action. To redo it:
```bash
pip install -r scripts/requirements.txt
python scripts/prep_photo.py source-image.png   # face-crop + bg removal
python scripts/make_ascii_svg.py                # -> ascii.svg
```
Tuning knobs (env vars): `FACE_UP`, `FACE_DOWN`, `FACE_WIDE` control the crop;
`NO_FACE=1` uses the whole subject.
